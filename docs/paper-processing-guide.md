# 论文处理工作流使用指南

## 概述
此工作流帮助您将学术论文PDF自动转换为结构化摘要、笔记和博客文章。

## 准备工作
1. 将论文PDF文件放入 `paper/<论文文件夹>/` 目录中
2. 确保PDF文件命名清晰（如 `论文标题.pdf`）

## 使用方法
运行以下命令处理特定论文：

```bash
node scripts/process-paper.js <论文文件夹名称>
```

示例：
```bash
node scripts/process-paper.js NeuPAN
```

## 处理步骤
1. **提取文本**：脚本从PDF提取文本并保存到 `extracted-text.txt`
2. **AI分析**：AI助手（opencode模型）分析提取的文本
3. **生成摘要**：AI助手创建 `summary.json`（JSON格式的结构化摘要）
4. **生成笔记**：AI助手创建 `notes.md`（详细的阅读笔记）
5. **生成博客**：AI助手创建博客文章到 `source/_posts/` 目录

## 生成的文件
处理完成后，以下文件将被创建：

1. **提取的文本** (`paper/<论文文件夹>/extracted-text.txt`)
   - PDF中提取的原始文本内容
   - 供AI助手分析使用

2. **JSON摘要** (`paper/<论文文件夹>/summary.json`)
   - 包含研究问题、方法论、核心发现和局限性
   - 格式符合AI助手分析模板

3. **Markdown笔记** (`paper/<论文文件夹>/notes.md`)
   - 详细的阅读笔记
   - 包含关键概念、方法详解、实验结果等

4. **博客文章** (`source/_posts/YYYY-MM-DD-<论文标题>.md`)
   - Hexo格式的博客文章
   - 包含Front Matter和结构化内容
   - 可直接部署到博客

## 自定义配置
- 修改博客文章的分类和标签以匹配个人需求
- 调整博客文章模板以满足个人风格

## 故障排除
- **PDF提取失败**: 确保PDF文件未加密且可读
- **文本编码问题**: 中文PDF可能需要额外处理
- **文件未生成**: 检查控制台输出中的错误信息
